### `em_etmeta_onload` (obuiapp_process)

```js
async (process, view) => {
  const GL_ITEM_PARAM = 'c_glitem_id';
  const HANDLER = 'org.openbravo.common.actionhandler.CheckAvailableCreditActionHandler';
  const form = view.theForm;
  const context = view.parentWindow.view.getContextInfo();
  const businessPartnerId = context.C_BPartner_ID || context.inpcBpartnerId;
  // The Currency parameter defaults to the BP currency (@BP_Currency_ID@). Its form value may
  // not be synced yet when onLoad runs, so fall back to the launching record's currency from the
  // context (keyed by inputName, e.g. inpbpCurrencyId — the same key onProcess reads). `null`
  // (not undefined) keeps the key in the JSON payload; the handler treats null/empty as
  // availableCredit:false. Classic always sends the key.
  const currencyId = form.getItem('C_Currency_ID').getValue() || context.inpbpCurrencyId || null;
  const glItem = form.getItem(GL_ITEM_PARAM);

  // TEMP diagnostic — confirm where the currency comes from at runtime, then remove this log.
  console.log('[SetNewCurrency.onLoad] currency resolution', {
    fieldValue: form.getItem('C_Currency_ID').getValue(),
    inpbpCurrencyId: context.inpbpCurrencyId,
    contextKeys: Object.keys(context),
    businessPartnerId: businessPartnerId,
    resolvedCurrencyId: currencyId,
  });

  const callback = (response, data) => {
    if (data.availableCredit) {
      view.messageBar.setMessage(isc.OBMessageBar.TYPE_WARNING, null, OB.I18N.getLabel('BPCurrencyChange'));
      glItem.show();
      glItem.setRequired(true);
    } else {
      glItem.hide();
    }
    form.redraw();
    view.handleButtonsStatus();
  };

  OB.RemoteCallManager.call(HANDLER, {
    businessPartnerId: businessPartnerId,
    currencyId: currencyId
  }, {}, callback);
}
```

### `em_etmeta_onprocess` (obuiapp_process)

```js
async (process, view) => {
  const form = view.theForm;
  const context = view.parentWindow.view.getContextInfo();
  const currencyFromId = context.inpbpCurrencyId;
  const currencyToId = form.getItem('C_Currency_ID').getValue();
  const glItemId = form.getItem('c_glitem_id').getValue();
  const setAmount = form.getItem('Amount').getValue();
  const currentBalance = context.inpsoCreditused;
  const foreignAmount = form.getItem('Foreign_Amount').getValue();

  if (currencyFromId !== currencyToId && glItemId && setAmount === true && currentBalance === 0 && foreignAmount !== 0) {
    const text = OB.I18N.getLabel('BPCurrencyChangeRate');
    view.messageBar.setMessage(isc.OBMessageBar.TYPE_ERROR, null, text);
    return { severity: 'error', text };
  }

  const response = await view.executeProcess();
  return response && response.message;
}
```

## Advisories (non-blocking)

1. **`view.executeProcess()` replaces the hand-built payload.** An earlier revision of this report
   reproduced `actionHandlerCall()` with `callAction('…SetNewBPCurrency', payload, …)` and a manually
   assembled payload — which depended on unverifiable top-level context keys. The current substrate
   provides `view.executeProcess()`, which builds the standard payload itself (record context +
   `_params`) and posts to the configured Java class. The manual `callAction` path is dropped.
2. **`onLoad` bare PK read.** Classic `onLoad` reads `getContextInfo().C_BPartner_ID`. The new
   `getContextInfo()` keys the launching record by `inputName`, so the bare `C_BPartner_ID` may be
   absent; the body reads `context.C_BPartner_ID || context.inpcBpartnerId` defensively. This drives only
   the credit-availability banner and the `G/L Item` visibility — non-fatal if the credit lookup yields
   no data. Manual test #1/#2 confirms it at runtime.
3. **textBoxStyle dropped.** Classic lines 32-34 append `'Required'` to `c_glitem_id.textBoxStyle`
   purely to render the required visual; in the new UI `setRequired(true)` drives that, so the
   style-string manipulation is redundant and was not migrated.
4. **`view` action methods need a controller.** `view.handleButtonsStatus()` is action-tier; it is live
   only when the modal injects a controller. QA must confirm it runs without
   `handleButtonsStatus is not implemented yet`.
5. **`setAmount === true`** — the Classic strict-equality check against boolean `true` is preserved
   verbatim. If the `Amount` (Set Amount) parameter does not surface a boolean in the new UI, test #4
   below will reveal it.
6. **Validation keys depend on launching-tab fields.** `inpbpCurrencyId` and `inpsoCreditused` ride
   `getContextInfo()` only because `BP_Currency_ID` and `SO_CreditUsed` are displayed fields on the
   Business Partner header tab (verified in `ad_field`, tab 220). If a future window restricts those
   fields, the validation guard would short-circuit. Test #4 covers this.

### Iteration 2 — `onLoad` payload (`#3` request, unblocks `#2`/`#4`)

With the crash gone, `onLoad` now reaches `OB.RemoteCallManager.call(CheckAvailableCreditActionHandler, …)`,
but the request body carried only `{ "businessPartnerId": … }`. The handler reads
`jsonData.getString("currencyId")` (`CheckAvailableCreditActionHandler.java:38`), which **throws
`JSONObject["currencyId"] not found`** when the key is absent (it tolerates a null/empty *value* →
`availableCredit:false`, lines 40-52). The `Currency` selector returns `undefined` when it has no value,
and `JSON.stringify` drops `undefined` keys, so `currencyId` never reached the server (Classic always
sends it). The error response made the callback fail, so the `availableCredit` branch never ran — which
is exactly why the warning (`#2`) didn't show and `G/L Item` (`#4`) never got hidden/shown.

| # | Fix |
|---|---|
| #3 (request) | **JS (`onLoad`).** Two parts: (a) keep the `currencyId` key in the payload (coerce to `null`, never `undefined`, which `JSON.stringify` drops); (b) source the value reliably. `getItem('C_Currency_ID').getValue()` correctly resolves to the `Currency` form key (`scriptProxies.ts:655-663`, default lands under `Currency` per `processDefaultsUtils.ts:172`), but the FIC default is **not synced into the form yet** when `onLoad` runs, so it reads `null` even when the BP has a currency. The launching record's currency is available in the context as `inpbpCurrencyId` (`getContextInfo()` spreads `parentRecord = buildPayloadByInputName(record, tab.fields)`), the same key `onProcess` reads — so: `const currencyId = form.getItem('C_Currency_ID').getValue() \|\| context.inpbpCurrencyId \|\| null;`. A temporary `console.log` was added to confirm the exact context key at runtime (remove once verified). Re-paste the updated `em_etmeta_onload` into `obuiapp_process.em_etmeta_onload`. |
| #2 | Resolved by #3 (request): with a successful response, `data.availableCredit === true` runs `messageBar.setMessage(BPCurrencyChange)`. |
| #4 | Resolved by #3 (request): the callback runs `glItem.hide()` (no credit) / `glItem.show()+setRequired` (credit). |
