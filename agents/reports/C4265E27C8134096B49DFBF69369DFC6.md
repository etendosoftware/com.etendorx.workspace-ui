## 3. Feasibility gate — BLOCKED

At least one **in-scope** mechanism is `unsupported`, so the conservative gate fires: **no migrated
code is generated** and the process stays `blocked`.

### 3.1 Primary blocker — `selectionChanged` `(record, state)` delta is not available

`onLoad` exists for the single purpose of installing the per-row selection handler:

```js
OB.ProductServices.onLoad = function (view) {
  var orderLinesGrid = view.theForm.getItem('grid').canvas.viewGrid;
  orderLinesGrid.selectionChanged = OB.ProductServices.relateOrderLinesSelectionChanged;
};
```

`relateOrderLinesSelectionChanged(record, state)` is a SmartClient **per-toggle** callback: it receives
the single record the user just clicked plus `state` (true = just selected, false = just deselected),
debounces per record id via `this.fireOnPause('selectionChanged' + record.id, …, 200)`, and chains
`this.Super('selectionChanged', arguments)`. Its downstream `doRelateOrderLinesSelectionChanged(record,
state, view)` is **fundamentally per-record-and-state**:

- it writes `relatedQuantity` on **that specific toggled record** — to `originalOrderedQuantity` when
  `state` is true (non-RFC window), or to `0` when deselected (L131-143);
- it forwards that exact `record` + `state` to `updateServicePrice`, whose error callback calls
  `orderLinesGrid.deselectRecord(record)` to roll back **that one record** (L171-173).

The new-UI chained `selectionChanged` subscriber is invoked as `fn(selection)` with the **full array of
currently-selected records** after the change — there is **no toggled `record` and no `state`**
(`WindowReferenceGrid.tsx:2676-2677`, subscriber typed `(selection: EntityData[]) => void`;
`scriptProxies.ts:485,873`). Reconstructing the `(record, state)` delta would require the substrate to
diff the previous selection against the new one and emit the change, which it does not do and which the
chained-subscriber contract does not promise. Emulating it inside the hook (keeping a previous-selection
snapshot in module scope) would change firing semantics and is not a faithful translation of a
framework-provided per-row callback — exactly the kind of behavior-altering workaround the gate
forbids.

Additionally:

- `this.fireOnPause(...)` is the **grid's** SmartClient debounce; the new-UI `fireOnPause` lives on the
  **view** proxy, and its per-record-id key depends on the missing `record`.
- `this.Super('selectionChanged', arguments)` has no equivalent and cannot be expressed.

### 3.2 Secondary issue — `activeView` / `parentView` context fallback (best-effort)

`updateServicePrice` reads the launching tab id via
`orderLinesGrid.view.parentWindow.activeView.getContextInfo(false, true, true, true)` and, when
`inpTabId` is empty, falls back to `…activeView.parentView.getContextInfo(...)` (L158-161). The
substrate exposes `view.getContextInfo()` / `view.parentWindow.view.getContextInfo()` (parent record
fields by `inputName` overlaid with current parameter values) but **not** an `activeView`/`parentView`
with a four-argument `getContextInfo`, and there is a single context surface, so the two-tier fallback
is not reproducible. This alone would be a non-blocking best-effort note; it is recorded here because
it compounds the primary blocker (the whole `selectionChanged → updateServicePrice` chain is the
process's core behavior).

## 5. What the platform would need (to unblock)

1. **Per-toggle `selectionChanged` delta.** Extend the chained grid `selectionChanged` subscriber to
   deliver the change delta in the Classic shape — invoke subscribers as `fn(record, state)` (the
   single record toggled + its new boolean), or expose a sibling subscription (e.g.
   `grid.onSelectionToggle((record, state) => …)`) — by diffing the previous selection in
   `WindowReferenceGrid` (`selectionChangedSubsRef`). Without the toggled record + state, the
   `relatedQuantity` write and the per-record `deselectRecord` rollback cannot be reproduced.
2. **Grid-relative `fireOnPause`.** Provide `fireOnPause` on the grid handle reachable inside the
   `selectionChanged` subscriber (the Classic `this`), so the per-record debounce key
   (`'selectionChanged' + record.id`) survives. (Soft requirement: a single trailing debounce on the
   delta callback would also satisfy parity.)
3. **`activeView` / `parentView` context accessor.** Give `view.parentWindow.activeView` (and its
   `parentView`) a `getContextInfo()` so the two-tier `if (!inpTabId)` fallback in `updateServicePrice`
   resolves the launching `inpTabId` the same way Classic does. (Best-effort; needed only for exact
   parity of the service-price recompute call.)
4. **(Out of scope for the migration columns, but required for full process parity)** A migration path
   for **grid-column field hooks** — `AD_FIELD.ONCHANGEFUNCTION` and `AD_FIELD.EM_OBUIAPP_VALIDATOR` —
   used inside an embedded window-reference grid (here `orderLinesGridQtyOnChange` and
   `QuantityValidate`). There is currently no `em_etmeta_*` column for per-grid-column onChange or
   validators, and `BigDecimal.multiply` (used by `orderLinesGridQtyOnChange`) is not implemented. Even
   if items 1-3 were solved, the cell-edit recompute (qty → amount/discount) and the quantity range
   validation would still be lost without this. See Advisory A1.

**Note on the editable column name.** The two grid-column field hooks bind on the **`relatedQuantity`**
column (`AD_FIELD_ID = 025D94512BBC49E3A941AB7114C5704E`, "Related Quantity"), so the migrated
`setColumnOnChange` / `setColumnValidator` register on `'relatedQuantity'` — **not** `'quantity'`. (An
earlier draft of this section wrongly said `'quantity'`; corrected here.)

**Note on `deselectRecord`.** The classic rollback `orderLinesGrid.deselectRecord(record)` passes a
**record object**, but the controller's `deselectRecord(index)` resolves by **numeric index**
(`WindowReferenceGrid.tsx:1279-1282` → `rows()[index]`). Passing the record verbatim would silently
no-op (lose the rollback). The migration therefore calls
`grid.deselectRecord(grid.getRecordIndex(record))` — a 1:1 translation using the proxy's own
index resolver, not a behavior change.

### U2 — Migration architecture decision

Because the live grid handle and all registration points (`onSelectionToggle`, `setColumnOnChange`,
`setColumnValidator`) are only reachable once the grid controller exists, **all behavior consolidates
into the "Pick/Edit Lines" parameter's `em_etmeta_on_grid_load`**, with the shared helpers in the
process-level `em_etmeta_payscript_logic` (module scope). The classic `onLoad` did nothing but install
`orderLinesGrid.selectionChanged = …`; that single side effect moves into the `onGridLoad` body as the
`grid.onSelectionToggle(...)` registration, so **`em_etmeta_onload` is left EMPTY**. This also sidesteps
the Section 6.5 caveat that `onLoad` does not run inside a nested modal — the grid hooks register from
`onGridLoad`, which always runs.

### U3 — Generated code per field

**`em_etmeta_payscript_logic`** — entity `obuiapp_process` (module body, ends in `return {…}`):

```js
const RFC_WINDOW_ID = 'FF808081330213E60133021822E40007';
const SERVICE_PRICE_HANDLER = 'org.openbravo.common.actionhandler.ServiceRelatedLinePriceActionHandler';

const ZERO = () => BigDecimal.prototype.ZERO;
const bd = (value) => new BigDecimal(String(value));

const grid = (view) => view.theForm.getItem('grid').canvas.viewGrid;

const updateTotalLinesAmount = (view) => {
  const form = view.theForm;
  const g = grid(view);
  const amountField = g.getFieldByColumnName('amount');
  const discountField = g.getFieldByColumnName('discountsAmt');
  const priceField = g.getFieldByColumnName('price');
  const relatedQtyField = g.getFieldByColumnName('relatedQuantity');
  const unitDiscountAmtField = g.getFieldByColumnName('unitDiscountsAmt');
  const selectedRecords = g.getSelectedRecords();

  let totalLinesAmt = ZERO();
  let totalLinesDiscountAmt = ZERO();
  let totalLinesPriceAmt = ZERO();
  let totalLinesRelatedQty = ZERO();
  let totalLinesUnitDiscountAmt = ZERO();

  selectedRecords.forEach((record) => {
    const index = g.getRecordIndex(record);
    totalLinesAmt = totalLinesAmt.add(bd(g.getEditedCell(index, amountField)));
    totalLinesDiscountAmt = totalLinesDiscountAmt.add(bd(g.getEditedCell(index, discountField)));
    totalLinesPriceAmt = totalLinesPriceAmt.add(bd(g.getEditedCell(index, priceField)));
    totalLinesRelatedQty = totalLinesRelatedQty.add(bd(g.getEditedCell(index, relatedQtyField)));
    totalLinesUnitDiscountAmt = totalLinesUnitDiscountAmt.add(bd(g.getEditedCell(index, unitDiscountAmtField)));
  });

  form.getItem('totallinesamount').setValue(Number(totalLinesAmt.toString()));
  form.getItem('totaldiscountsamount').setValue(Number(totalLinesDiscountAmt.toString()));
  form.getItem('totapriceamount').setValue(Number(totalLinesPriceAmt.toString()));
  form.getItem('totalrelatedqty').setValue(Number(totalLinesRelatedQty.toString()));
  form.getItem('totalUnitDiscountsAmt').setValue(Number(totalLinesUnitDiscountAmt.toString()));
};

const buildRelatedLinesInfo = (g) => {
  const info = {
    lineAmount: [],
    lineDiscounts: [],
    linePriceamount: [],
    lineRelatedqty: [],
    lineUnitdiscountsamt: [],
    relatedLines: []
  };
  g.getSelectedRecords().forEach((selectedRecord) => {
    info.lineAmount.push(g.getEditedCell(selectedRecord, g.getFieldByColumnName('amount')));
    info.lineDiscounts.push(g.getEditedCell(selectedRecord, g.getFieldByColumnName('discountsAmt')));
    info.linePriceamount.push(g.getEditedCell(selectedRecord, g.getFieldByColumnName('price')));
    info.lineRelatedqty.push(g.getEditedCell(selectedRecord, g.getFieldByColumnName('relatedQuantity')));
    info.lineUnitdiscountsamt.push(g.getEditedCell(selectedRecord, g.getFieldByColumnName('unitDiscountsAmt')));
    info.relatedLines.push(selectedRecord.id);
  });
  return info;
};

const updateServicePrice = (view, record, state) => {
  const form = view.theForm;
  const g = grid(view);
  const totalServiceAmount = form.getItem('totalserviceamount');
  const recordId = record ? record.id : null;

  let contextInfo = view.parentWindow.activeView.getContextInfo();
  if (!contextInfo.inpTabId) {
    contextInfo = view.parentWindow.activeView.parentView.getContextInfo();
  }

  const parameters = {
    orderlineId: form.getItem('orderlineId').getValue(),
    amount: form.getItem('totallinesamount').getValue(),
    discounts: form.getItem('totaldiscountsamount').getValue(),
    priceamount: form.getItem('totapriceamount').getValue(),
    relatedqty: form.getItem('totalrelatedqty').getValue(),
    unitdiscountsamt: form.getItem('totalUnitDiscountsAmt').getValue(),
    orderLineToRelateId: recordId,
    tabId: contextInfo.inpTabId,
    state: state,
    relatedLinesInfo: buildRelatedLinesInfo(g)
  };

  OB.RemoteCallManager.call(SERVICE_PRICE_HANDLER, parameters, {}, (response, data) => {
    if (data.amount || data.amount === 0) {
      totalServiceAmount.setValue(Number(data.amount));
      if (data.message) {
        view.messageBar.setMessage(isc.OBMessageBar.TYPE_WARNING, data.message.title, data.message.text);
      }
    } else {
      view.messageBar.setMessage(isc.OBMessageBar.TYPE_ERROR, data.message.title, data.message.text);
      if (record) {
        g.deselectRecord(g.getRecordIndex(record));
      }
    }
  });
};

const relatedQuantityOnSelect = (view, record, state) => {
  const g = grid(view);
  const index = g.getRecordIndex(record);
  if (view.windowId !== RFC_WINDOW_ID && state) {
    g.setEditValue(index, 'relatedQuantity', Number(bd(record.originalOrderedQuantity).toString()));
  } else {
    g.setEditValue(index, 'relatedQuantity', 0);
  }
  updateTotalLinesAmount(view);
  updateServicePrice(view, record, state);
};

const quantityCellOnChange = (item, view, form, g) => {
  const index = g.getRecordIndex(item.record);
  const newQty = bd(item.getValue());
  const precision = form.getItem('pricePrecision').getValue();
  const newAmount = newQty.multiply(bd(item.record.price)).setScale(precision, BigDecimal.prototype.ROUND_HALF_UP);
  const newDiscount = newQty.multiply(bd(item.record.unitDiscountsAmt)).setScale(precision, BigDecimal.prototype.ROUND_HALF_UP);
  g.setEditValue(index, 'amount', Number(newAmount.toString()));
  g.setEditValue(index, 'discountsAmount', Number(newDiscount.toString()));
  updateTotalLinesAmount(view);
  updateServicePrice(view, item.record, null);
};

const isNumber = (value) => typeof value === 'number' && !isNaN(value);

const quantityValidate = (item, validator, value, record) => {
  if (!isNumber(value)) {
    return false;
  }
  const quantity = bd(value);
  const returnedQty = bd(record.returnQtyOtherRM);
  const recordQty = bd(record.originalOrderedQuantity);
  const windowId = item.grid.view.windowId;
  const messageBar = item.grid.view.messageBar;
  const isRfc = windowId === RFC_WINDOW_ID;

  if (isRfc && (value < 0 || quantity.compareTo(recordQty.subtract(returnedQty)) > 0)) {
    messageBar.setMessage(isc.OBMessageBar.TYPE_ERROR, null, OB.I18N.getLabel('OBUIAPP_RM_OutOfRange', [recordQty.subtract(returnedQty).toString()]));
    return false;
  }
  if (!isRfc && recordQty.compareTo(ZERO()) < 0 && (quantity.compareTo(recordQty) < 0 || value > 0)) {
    messageBar.setMessage(isc.OBMessageBar.TYPE_ERROR, null, OB.I18N.getLabel('ServiceQuantityLessThanOrdered', [recordQty.toString()]));
    return false;
  }
  if (!isRfc && recordQty.compareTo(ZERO()) > 0 && (quantity.compareTo(recordQty) > 0 || value < 0)) {
    messageBar.setMessage(isc.OBMessageBar.TYPE_ERROR, null, OB.I18N.getLabel('ServiceQuantityMoreThanOrdered', [recordQty.toString()]));
    return false;
  }
  return true;
};

return {
  updateTotalLinesAmount,
  updateServicePrice,
  relatedQuantityOnSelect,
  quantityCellOnChange,
  quantityValidate
};
```
