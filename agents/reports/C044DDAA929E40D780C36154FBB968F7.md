## Advisories (non-blocking)

1. **Dummy criterion dropped.** The classic line
   `newCriteria.criteria.push(isc.OBRestDataSource.getDummyCriterion())` exists only to force the
   SmartClient datasource to re-fetch. In the new UI `invalidateCache()` already forces a re-query, so
   the push is unnecessary. The `newCriteria.criteria = newCriteria.criteria || []` line is kept to
   preserve the original semantics of normalizing the criteria array before invalidating the cache,
   even though the resulting object is no longer pushed into.
2. **`newCriteria` is computed for parity.** `addSelectedIDsToCriteria(getCriteria(), true)` is
   retained because in the classic flow it captures the currently selected order IDs so the selection
   survives the re-fetch. If manual QA shows the selection is already preserved by the new grid's
   invalidate/refetch and this call is redundant, it can be simplified to just `orderGrid.invalidateCache(); form.redraw();`.
3. **No cloning needed.** This file is self-contained and not part of the `processRecords` family or any
   namespace-sharing cluster.
