## Classic file structure (what the file actually does)

`ob-aprm-matchStatement.js` (377 lines) declares the `OB.APRM.MatchStatement` namespace and the
SmartClient row component `APRMMatchStatGridButtonsComponent`. It attaches behavior through **two
distinct binding channels**:

### Channel A — process hooks (bound via `obuiapp_process` columns)

| Classic binding | Function | What it does |
|---|---|---|
| `on_load_function` | `OB.APRM.MatchStatement.onLoad` | Hides Cancel + the popup close `X`; finds the `UN` footer button by `_buttonValue` and **hides it + reassigns its `.action`** to call `UnMatchSelectedTransactionsActionHandler` with the grid selection; runs the matching algorithm via `MatchStatementOnLoadActionHandler` behind an `isc.confirm`; shows a "persist info" banner (with a "never show again" link → `addPreference`) via `MatchStatementOnLoadGetPreferenceActionHandler`. |
| `clientsidevalidation` | `OB.APRM.MatchStatement.onProcess` | Calls `actionHandlerCall()` unconditionally (no validation — just proceed). |
| `on_refresh_function` | `OB.APRM.MatchStatement.onRefresh` | `grid.invalidateCache()` + `view.theForm.redraw()`. |

The two process parameters (`Actions` = Button List with `OK` / `RE` Reconcile All / `UN` Unmatch
Selected; `Match Statement` = Window Reference grid) carry **no** classic
`onchangefunction` / `ongridloadfunction`.

### Channel B — per-row grid component (bound via `AD_FIELD.CLIENTCLASS`)

`APRMMatchStatGridButtonsComponent` is registered on the grid's **`Match`** column via
`AD_FIELD.CLIENTCLASS` (tab "Match Statement"). It renders three inline icon-buttons per row
(`search` / `add` / `clearRight`), enables/disables them by `record.cleared`, calls
`OB.APRM.MatchStatement.selectionChanged(grid, record)` on init to drive the `UN` button's
show/hide, and on `click` does `grid.selectSingleRecord(record)`. The `search`/`add` buttons each
pre-check `CheckRecordChangedActionHandler` then `standardWindow.openProcess(...)` a nested process
(`154CB4F9…` Find Transactions to Match, `E68790A7…` Add Transaction); `clearRight` calls
`UnMatchTransactionActionHandler`.

### `em_etmeta_onload` — entity `obuiapp_process`

```js
async (process, view) => {
  const UNMATCH_BUTTON_VALUE = 'UN';
  const UNMATCH_HANDLER =
    'org.openbravo.advpaymentmngt.actionHandler.UnMatchSelectedTransactionsActionHandler';
  const ALGORITHM_HANDLER =
    'org.openbravo.advpaymentmngt.actionHandler.MatchStatementOnLoadActionHandler';
  const GET_PREFERENCE_HANDLER =
    'org.openbravo.advpaymentmngt.actionHandler.MatchStatementOnLoadGetPreferenceActionHandler';
  const ADD_PREFERENCE_HANDLER =
    'org.openbravo.advpaymentmngt.actionHandler.MatchStatementOnLoadPreferenceActionHandler';

  const grid = view.theForm.getItem('match_statement').canvas.viewGrid;

  const showResultMessage = function(data) {
    if (!data || !data.message) {
      return;
    }
    const severity = data.message.severity;
    if (severity === 'error') {
      view.messageBar.setMessage(isc.OBMessageBar.TYPE_ERROR, data.message.title, data.message.text);
    } else if (severity === 'success') {
      view.messageBar.setMessage(isc.OBMessageBar.TYPE_SUCCESS, data.message.title, data.message.text);
    } else if (severity === 'warning') {
      view.messageBar.setMessage(isc.OBMessageBar.TYPE_WARNING, data.message.title, data.message.text);
    }
  };

  view.cancelButton.hide();
  view.parentElement.parentElement.closeButton.hide();

  const buttons = view.popupButtons.members;
  let unmatchButton;
  for (let i = 0; i < buttons.length; i++) {
    if (buttons[i]._buttonValue === UNMATCH_BUTTON_VALUE) {
      unmatchButton = buttons[i];
      break;
    }
  }
  if (unmatchButton) {
    view.unmatchButton = unmatchButton;
    unmatchButton.hide();
    unmatchButton.action = function() {
      OB.RemoteCallManager.call(
        UNMATCH_HANDLER,
        { bankStatementLineIds: grid.getSelectedRecords() },
        {},
        function(response, data) {
          view.onRefreshFunction(view);
          showResultMessage(data);
        }
      );
    };
  }

  const addPreference = function() {
    OB.RemoteCallManager.call(ADD_PREFERENCE_HANDLER, {}, {}, function() {});
  };

  const runAlgorithm = function(ok) {
    const params = { executeMatching: ok };
    if (view.sourceView && view.sourceView.getContextInfo) {
      params.context = view.sourceView.getContextInfo();
    }
    OB.RemoteCallManager.call(ALGORITHM_HANDLER, {}, params, function(response, data) {
      if (data && data.responseActions) {
        OB.Utilities.Action.executeJSON(data.responseActions, null, null, view);
      }
      grid.invalidateCache();
    });
  };

  OB.RemoteCallManager.call(GET_PREFERENCE_HANDLER, {}, {}, function(response, data) {
    if (data && !data.preference) {
      view.messageBar.setMessage(
        isc.OBMessageBar.TYPE_INFO,
        null,
        OB.I18N.getLabel('APRM_GRID_PERSIST_MESSAGE'),
        [
          {
            label: OB.I18N.getLabel('OBUIAPP_NeverShowMessageAgain'),
            onClick: function() {
              view.messageBar.hide();
              addPreference();
            }
          }
        ]
      );
    }
  });

  isc.confirm(OB.I18N.getLabel('APRM_AlgorithmConfirm'), runAlgorithm);
}
```

  const FIND_PROCESS_ID = '154CB4F9274A479CB38A285E16984539';
  const ADD_PROCESS_ID = 'E68790A7B65F4D45AB35E2BAE34C1F39';
  const CHECK_CHANGED_HANDLER =

  const UNMATCH_TRANSACTION_HANDLER =
    'org.openbravo.advpaymentmngt.actionHandler.UnMatchTransactionActionHandler';
  const SEARCH_LABEL = 'APRM_MATCHTRANSACTION_SEARCH_BUTTON';
  const ADD_LABEL = 'APRM_MATCHTRANSACTION_ADD_BUTTON';
  const DELETE_LABEL = 'APRM_MATCHTRANSACTION_DELETE_BUTTON';

  const showError = function(data) {
    if (data && data.message && data.message.severity === 'error') {
      view.messageBar.setMessage(isc.OBMessageBar.TYPE_ERROR, data.message.title, data.message.text);
      return true;
    }
    return false;
  };

  const recordUpdated = function(record) {
    const updated = new Date();
    if (record.bslUpdated) {
      updated.setTime(new Date(record.bslUpdated).getTime());
    }
    return updated;
  };

  const refreshUnmatchVisibility = function() {
    const selection = grid.getSelectedRecords() || [];
    let hasCleared = false;
    for (let i = 0; i < selection.length; i++) {
      if (selection[i] && selection[i].cleared) {
        hasCleared = true;
        break;
      }
    }
    if (!view.unmatchButton) {
      return;
    }
    if (hasCleared) {
      view.unmatchButton.show();
    } else {
      view.unmatchButton.hide();
    }
  };

  const openMatchProcess = function(record, processId, label) {
    OB.RemoteCallManager.call(
      CHECK_CHANGED_HANDLER,
      { bankStatementLineId: record.id, updated: recordUpdated(record) },
      {},
      function(response, data) {
        view.onRefreshFunction(view);
        if (showError(data)) {
          return;
        }
        view.openProcess({
          paramWindow: true,
          processId: processId,
          windowId: view.windowId,
          externalParams: {
            bankStatementLineId: record.id,
            transactionDate: record.transactionDate
          },
          windowTitle: OB.I18N.getLabel(label)
        });
      }
    );
  };

  const clearTransaction = function(record) {
    OB.RemoteCallManager.call(
      UNMATCH_TRANSACTION_HANDLER,
      { bankStatementLineId: record.id, updated: recordUpdated(record) },
      {},
      function(response, data) {
        view.onRefreshFunction(view);
        showError(data);
      }
    );
  };

  grid.setRowActions(function(ctx) {
    const record = ctx.record;
    return {
      buttons: [
        {
          icon: 'search',
          prompt: OB.I18N.getLabel(SEARCH_LABEL),
          disabled: !!record.cleared,
          action: function() {
            openMatchProcess(record, FIND_PROCESS_ID, SEARCH_LABEL);
          }
        },
        {
          icon: 'add',
          prompt: OB.I18N.getLabel(ADD_LABEL),
          disabled: !!record.cleared,
          action: function() {
            openMatchProcess(record, ADD_PROCESS_ID, ADD_LABEL);
          }
        },
        {
          icon: 'clearRight',
          prompt: OB.I18N.getLabel(DELETE_LABEL),
          disabled: !record.cleared,
          action: function() {
            clearTransaction(record);
          }
        }
      ]
    };
  });

  grid.onSelectionToggle(function(record, state) {
    refreshUnmatchVisibility();
  });
}

## Advisories

1. **`UN` action routing.** The Classic `button.action = fn` is ported verbatim onto the
   `view.popupButtons.members` handle whose `_buttonValue === 'UN'`. At click time the closure reads the
   live `grid.getSelectedRecords()` and posts them to `UnMatchSelectedTransactionsActionHandler`; the
   standard submit is bypassed for that button. `OK` and `RE` (Reconcile All) keep the standard submit to
   `MatchStatementActionHandler` — unchanged.
2. **Persist-info banner link → structured `actions`.** The Classic inline
   `<a onclick="…hide(); addPreference()">` (which the sanitizer drops) is ported as a structured
   `actions: [{ label, onClick }]` entry that calls `view.messageBar.hide()` then posts
   `MatchStatementOnLoadPreferenceActionHandler`. Title passed as `null`; the label text is the
   `OBUIAPP_NeverShowMessageAgain` label. Verify the link affordance renders and persists the preference.
3. **Algorithm-before-first-fetch ordering (semantic difference, accepted).** Classic suspends the grid
   datasource (`grid.dataSource = null` / `dataSourceOrig` / `filterByEditor`) so the matching algorithm
   (`MatchStatementOnLoadActionHandler`) runs *before* the first row fetch, then restores it. The new
   grid always re-queries and there is no `onLoad`-level hook to defer a Window-Reference grid's first
   fetch, so the migrated `onLoad` runs the algorithm behind the `isc.confirm` and calls
   `grid.invalidateCache()` in the callback to re-pull after the algorithm resolves. The user-visible end
   state (rows reflect the algorithm result) is the same; the exact fetch *ordering* on open is not
   byte-identical. **This is the item to watch most closely in QA** (test 2/3).
4. **`selectSingleRecord` on row-button click not reproduced (minor).** The declarative row-action click
   never toggles selection (doc 8.4), so the Classic side effect (clicking a row icon-button selects that
   single row) is dropped. The icon-button still performs its own action. Low impact.
5. **`responseActions` from the algorithm.** `MatchStatementOnLoadActionHandler` may return
   `responseActions`; they are dispatched via `OB.Utilities.Action.executeJSON(...)` exactly as Classic.
6. **`bslUpdated` parsing.** Classic reads `record.bslUpdated.getTime()` (a SmartClient Date). The new-UI
   datasource may deliver it as an ISO string, so the migrated `recordUpdated` wraps it in `new Date(...)`
   defensively before `getTime()`. Confirm the `CheckRecordChangedActionHandler` optimistic-lock check
   still passes (test 5/6) — if the column arrives in a non-ISO shape, the `updated` value may need a
   format tweak.
