## Why it is blocked (technical detail)

1. **No hook surface.** The migration target has exactly five entry points (onLoad, onProcess,
   onRefresh, onChange, onGridLoad). This process exposes **none** — no hook columns, no parameters.
   Its whole behavior lives in a **button handler** (`OB.OpenClose.openClose`) that the new-UI
   process-dialog runtime never invokes; the new UI opens a parameter form, not this custom popup.

2. **Bespoke `isc.OBPopup`, not a parameter form.** The supported render paths are the standard
   metadata form or a custom component built from a **schema returned by `onLoad`**
   (`etmetaCustomComponent=true`). Arbitrary SmartClient widget construction
   (`isc.DynamicForm`/`isc.VLayout`/`isc.OBFormButton`/`isc.OBPopup`) is not in the capability
   catalog. Even the custom-component path is unreachable here: there is no `onLoad` hook wired, and
   the doc defines no schema shape for a "single dynamic-combo + OK/Cancel" dialog.

3. **The dynamic "Action" combo has no metadata home.** A faithful version would require a net-new
   `OBUIAPP_Parameter` (a list/select), its value map seeded at open via the `ACTION_COMBO`
   round-trip, plus carrying `closePeriodStepId`. That is net-new metadata authoring and a parameter
   that does not exist — not a translation of existing JS — and writing the DB is out of contract.

4. **Grid cell renderers out of scope.** `PeriodControlStatus_Field` / `PeriodStatus_Field` are
   `AD_FIELD.CLIENTCLASS` display components on the C_Period / C_PeriodControl grids, not process
   hooks; they have no `em_etmeta_*` channel.

**Substrate note (`em_etmeta_onprocess` dead path).** Even setting aside the above, this process
has no `clientsidevalidation`, so there is nothing to place in `em_etmeta_onprocess`; and for a
`uipattern=M` process the onprocess path is not the entry point. The block is broader than the
onprocess dead-path: there is no metadata-hook surface at all.

## What the platform would need to unblock this

- A documented **custom-component schema** for an action-picker dialog (single dynamic-populated
  select + OK/Cancel) returned by `onLoad` under `etmetaCustomComponent=true`, OR
- A supported way to declare a **list/select parameter** whose value map (and a hidden
  `closePeriodStepId`) is seeded at dialog open from a server `ACTION_COMBO` round-trip, so the
  behavior could be expressed as `onLoad` (fetch combo) + `onProcess` (`view.executeProcess` /
  `callAction(OpenClosePeriodHandler, { closePeriodStepId, recordIdList, action })`). This also
  requires authoring the missing parameter as metadata (a human/DB task, outside Babel's contract).
- (Independently) a migration channel for `AD_FIELD.CLIENTCLASS` grid cell renderers, if the colored
  period-status cells are to be reproduced.
