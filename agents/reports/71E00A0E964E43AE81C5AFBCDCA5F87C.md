  const CURRENCY_FIELD = 'C_Currency_ID';

  };

}

## Advisories (non-blocking)

1. **Namespace dropped.** The classic `OB.ETRVS.OnChange.*` namespace was a wrapper for one
   handler with no shared helpers; it is unnecessary in the new UI. No `em_etmeta_payscript_logic`.
2. **`valueMap` direct mutation â†’ `setValueMap`/`getValueMap`.** Classic mutated
   `item.valueMap[id] = identifier` in place. The proxy has no live `valueMap` property; the faithful
   equivalent reads the current map, replaces/adds the currency entry, and writes it back. Behavior
   (the currency selector shows the resolved currency identifier) is preserved. `getValueMap()` returns
   `ListOption[]` (`{ id, value, label }`); `setValueMap` normalizes that shape.
3. **Cross-module handler reuse.** The handler `AddPaymentOrganizationActionHandler` lives in
   `org.openbravo.advpaymentmngt`; the classic Valued Stock Report already reused it. Migration keeps
   the same handler â€” no change of behavior.
4. **`form.redraw()` kept for parity** though it is a reactive no-op in the new UI.
5. **No `qa-passed` reference available.** The inventory currently has no `qa-passed` process, so no
   trusted few-shot reference was used. Patterns follow `new-javascript-code.md` directly.
