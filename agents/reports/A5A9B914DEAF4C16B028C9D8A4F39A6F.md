  `ScriptHookContext.getValueMap` type (`=> ListOption[]`) and `readValueMap`. `setValueMap`
  normalizes either an array or a plain `{ id: label }` object
  (`normalizeValueMapEntries` / `applyValueMap`).

## Generated code per field

### `em_etmeta_onload` (process `obuiapp_process`)

```js
async (process, view) => {
  const TABLE_ID = "318";
  const HANDLER = "com.smf.jobs.defaults.ProcessInvoicesDefaults";
  const form = view.theForm;
  const docActionField = form.getItem("DocAction");
  const selectedRecords = view.selectedRecords || [];
  const isProcessing = "";

  const documentStatuses = [];
  const documentActions = [];
  for (const record of selectedRecords) {
    if (!documentStatuses.includes(record.documentStatus)) {
      documentStatuses.push(record.documentStatus);
    }
    if (!documentActions.includes(record.documentAction)) {
      documentActions.push(record.documentAction);
    }
  }

  form.getItem("VoidDate").hide();
  form.getItem("VoidAccountingDate").hide();
  form.getItem("POReference").hide();

  OB.RemoteCallManager.call(
    HANDLER,
    {
      documentStatuses: documentStatuses,
      isProcessing: isProcessing,
      tableId: TABLE_ID,
    },
    {},
    (response, data, request) => {
      const actions = data.actions || [];
      const currentValues = docActionField.getValueMap();
      const newValues = [];

      for (const action of actions) {
        const match = currentValues.find((o) => o.value === action || o.id === action);
        if (match) {
          newValues.push(match);
        }
      }

      docActionField.setValueMap(newValues);

      const firstAction = documentActions[0];
      const hasFirstAction = newValues.some((o) => o.value === firstAction || o.id === firstAction);
      if (hasFirstAction) {
        docActionField.setValueProgrammatically(firstAction);
      } else {
        docActionField.setValueProgrammatically(docActionField.getFirstOptionValue());
      }
    }
  );
}
```

Compile-check: PASS (evaluates to a function).

### `Document Action` parameter (`obuiapp_parameter`, col `DocAction`) → `em_etmeta_on_parameter_change`

```js
(item, view, form, grid) => {
  const VOID_ACTIONS = ["VO", "RC"];
  const INVOICE_WINDOW_ID = "183";
  const docActionField = form.getItem("DocAction");
  const selectedRecords = view.parentWindow.view.viewGrid.getSelectedRecords();
  const isVoidAction = VOID_ACTIONS.includes(docActionField.getValue());

  if (view.windowId === INVOICE_WINDOW_ID && isVoidAction) {
    form.getItem("VoidDate").show();
    form.getItem("VoidAccountingDate").show();
    if (selectedRecords.length < 2) {
      form.getItem("POReference").show();
    }
  }
}
```

## Advisories (non-blocking observations)

- **`POReference` does not exist on this process.** Classic `ProcessInvoices.onLoad` hides
  `form.getItem('POReference')` and `onChangeDocumentAction` shows it for window `183` with
  `< 2` records. This process declares only `DocAction`, `VoidDate`, `VoidAccountingDate` —
  no `POReference`. In Classic, `getItem('POReference')` on a process without that field
  returns null and `.hide()/.show()` is a no-op; the new UI behaves the same (proxy with no
  backing field). Carried over **verbatim**; it is inert here. The `POReference` show path
  is only meaningful for the sibling processes that declare it.
- **SmartClient `Array.prototype.add` → native `.push`.** Classic lines 21/24 use
  `documentStatuses.add(...)` / `documentActions.add(...)`, a SmartClient Array extension.
  Migrated as native `.push(...)`.
- **`getValueMap()` shape change.** New-UI `getValueMap()` returns a `ListOption[]` array,
  not the Classic `{ id: label }` object. The Classic membership lookup `currentValues[action]`
  is ported to `.find(o => o.value === action || o.id === action)`, and `setValueMap` is fed
  the matched `ListOption[]` (the substrate normalizes either form). The default-selection
  branch (`newValues[documentActions[0]]`) is ported to `newValues.some(o => o.value === firstAction || o.id === firstAction)`.
- **onLoad selection accessor.** Classic onLoad reads
  `view.parentWindow.view.viewGrid.getSelectedRecords()`; per the equivalence table the onLoad
  hook uses `view.selectedRecords` (the launch-context selection). The **onChange** hook keeps
  the literal Classic DOM walk, which is an alias of the same accessor.
- **Cloning implication (NOT actioned in this run).** `processRecords.js` backs 5 processes
  (`A5A9B914` ProcessInvoices, plus `B4A21A61`, `8DF818E4`, `33338B1F`, `272C8D38` —
  ProcessOrders / ProcessShipment / Create Inverse Order). The same two now-supported methods
  unblock the **whole** family, and each member's `onLoad` differs only in the `*Defaults`
  handler / `tableId` / `tabId` (and Orders/Shipment additionally use `view.okButton.enable()` /
  `isEnabled()`, which the doc documents as supported). Per the user's scope, **only A5A9B914
  is migrated here.** The sibling rows are intentionally left untouched and should be cloned
  in their own runs.
