### Why the `'800000'` section toggle is not a capability gap

In Classic, `sec = form.getItem('800000')` returned the SmartClient **section** widget for the
"Dimensions" field group, and `sec.show()/hide()` toggled the whole section header along with its
fields. In the new UI:

- Sections are not free-floating widgets. `groupProcessParametersByFieldGroup(visibleParameters)`
  (`components/ProcessModal/utils/groupProcessParametersByFieldGroup.ts`) builds a `CollapsibleSection`
  per field group **from the parameters that are currently renderable**.
- A parameter is hidden by `item.hide()` writing `'<name>.display' = false` into the script-logic
  store; the scalar selector (`ProcessParameterSelector`, reads `logicFields['<name>.display']`) and
  `isParameterRenderable` both honor that flag.
- When all seven Dimensions parameters are hidden, the group has no renderable members, so no
  `CollapsibleSection` is produced for it — the section header disappears on its own. When they are
  shown again, the section reappears.

So toggling the member parameters reproduces the exact visible outcome of the Classic `sec.show()/hide()`.
A literal `form.getItem('800000').hide()/show()` would resolve to an item proxy bound to the unmatched
key `'800000'` and write `'800000.display'`, which no rendered element reads — a harmless no-op. It is
dropped to keep the body honest and free of dead writes; behavior is identical with or without it.

```js
(item, view, form) => {
  const DIMENSION_FIELDS = [
    "FromAccount",
    "ToAccount",
    "BusinessPartner",
    "product",
    "c_project",
    "groupBy",
    "includeZeroFigures",
  ];
  const SUMMARY_LEVEL = "S";

  const setVisibility = (fieldNames, visible) => {
    fieldNames.forEach((fieldName) => {
      const field = form.getItem(fieldName);
      if (field) {
        if (visible) {
          field.show();
        } else {
          field.hide();
        }
      }
    });
  };

  const accountLevelItem = form.getItem("AccountLevel");
  const accountLevel = accountLevelItem ? accountLevelItem.getValue() : null;

  setVisibility(DIMENSION_FIELDS, accountLevel === SUMMARY_LEVEL);
}
```

## Advisories (non-blocking)

1. **Field-group section toggle dropped (behavior preserved).** The classic `showHideDimensions` ends
   with `sec = form.getItem('800000'); sec.show()/hide()`, where `800000` is the **AD_FieldGroup**
   "Dimensions", not a parameter. The new UI has no section widget addressable by that id; sections are
   derived by grouping visible parameters. Hiding/showing the seven Dimensions parameters already
   makes the whole section disappear/appear, so the explicit `sec` toggle is redundant and was dropped.
   A literal port would be a harmless no-op (`'800000.display'` is read by nothing). Net effect is
   identical to Classic. This is the only line of the classic function that is not ported 1:1.

2. **`showHideDimensions` migrated from scratch.** This is the first ETFRA process whose in-scope hook
   is `showHideDimensions`; the three sibling ETFRA reports migrated other ETFRA functions
   (`showDatesFieldsOnChange`, `organizationOnChange`, `showDocumentNoField`) and explicitly left
   `showHideDimensions` as dead code. No verbatim cluster clone existed, so no cluster-cloning reuse
   applied here.

3. **Refactor vs. line-for-line.** The classic callback uses one big if/else calling `.show()`/`.hide()`
   on each field individually. The migrated body keeps identical behavior but factors the field list
   into a `DIMENSION_FIELDS` array plus a `setVisibility` helper and a single `accountLevel === 'S'`
   comparison, reducing repetition and cognitive complexity. The shown/hidden set and the `'S'` branch
   are unchanged.

4. **Null-guard added.** The classic code assumes `getItem(...)` always returns a widget (it only
   guards `AccountLevel`). The migrated body guards every lookup (`if (field)`), so a missing/renamed
   field degrades gracefully instead of throwing inside the callback. Functionally equivalent for the
   present field set.

5. **List value compared as string.** `AccountLevel` is a List reference (`ad_reference_id = 17`), so
   `getValue()` returns the raw list value (`'S'` for *Summary*) — a string. The comparison
   `accountLevel === 'S'` matches Classic exactly; no numeric coercion applies here (numeric coercion
   in the item proxy only affects numeric references).

6. **No namespace bootstrap needed.** The classic file starts with `OB.ETFRA = {}` /
   `OB.ETFRA.OnChange = {}` to register the namespace. In the new UI each hook owns its scope, so no
   self-registration is required for this single self-contained `onChange`; `em_etmeta_payscript_logic`
   stays empty.
