## Generated code per field

### `obuiapp_parameter` → **Organization** (`AD_Org`, `72ED21539E4541DF91D6EC779373451F`) → `em_etmeta_on_parameter_change`

```js
(item, view, form) => {
  const CHANGE_ACCT_SCHEMA_HANDLER =
    "com.etendoerp.financial.reports.advanced.handler.ChangeAcctSchemaActionHandler";
  const GENERAL_LEDGER_FIELD = "GL_Journal_ID";

  const organization = item.getValue() ? item.getValue() : "";

  const setAcctSchema = (response, data) => {
    const generalLedger = form.getItem(GENERAL_LEDGER_FIELD);
    if (generalLedger) {
      generalLedger.setValueFromRecord({
        id: data.acctSchema,
        _identifier: data.acctSchemaIdIdentifier,
      });
    }
  };

  if (organization !== "") {
    OB.RemoteCallManager.call(
      CHANGE_ACCT_SCHEMA_HANDLER,
      { organization: organization },
      {},
      setAcctSchema
    );
  }
}
```

Compile-check: evaluates to a function, arity 3 — OK. This is the cluster-cloned body, identical to the
one delivered for `56E951BB13A44AFBB642291081613E46`.

```js
(item, view, form) => {
  const DOCUMENT_NO_FIELD = "DocumentNo";
  const SHOW_DOCUMENT_NO_FIELD = "showDocumentNo";
  const DOCUMENT_TYPES_WITHOUT_DOC_NO = [
    "361",
    "304",
    "305",

    "306",

    "8D951A5E22934A819342B97E5755D063",
    "12CD29ED0686428DACEF24B5040A8EA5",
    "21C51C14E7E14F05AE98A0DCFF569C66",

    "302",
    "257F25CAB32A4009AA45EFCE2ADE5F1C",
    "88BE65DDD6064F25A1837009F7375210",
  ];

  const selectedTypes = item.getValue();
  const documentNo = form.getItem(DOCUMENT_NO_FIELD);
  const showDocumentNo = form.getItem(SHOW_DOCUMENT_NO_FIELD);

  const getSelectedValue = (values) =>
    values && typeof values.get === "function" ? values.get(0) : values && values[0];

  const isSingleSupportedType =
    selectedTypes &&
    selectedTypes.length === 1 &&
    DOCUMENT_TYPES_WITHOUT_DOC_NO.includes(getSelectedValue(selectedTypes));

  if (isSingleSupportedType) {
    showDocumentNo.setValue(false);
    documentNo.setDisabled(false);
  } else {
    showDocumentNo.setValue(true);
    documentNo.setDisabled(true);
    documentNo.clearValue();
  }
}
```

## Advisories (non-blocking)

1. **Two in-scope hooks, not one.** Unlike its ETFRA siblings (D37588 binds only `showDatesFieldsOnChange`,
   56E951BB binds only `organizationOnChange`), this process binds **two** onChange functions. The
   cluster-cloning advisory hint (56E951BB #7) covered only `organizationOnChange`; the independent
   binding check surfaced the second hook (`showDocumentNoField` on **Document Type**), which is unique to
   this process. Both are migrated.
2. **Cluster cloning — `organizationOnChange` reused verbatim.** The **Organization** body is the exact
   same migrated code delivered for 56E951BB (ETFRA namespace-sharing cluster). A future fix to this logic
   must be applied to each member row independently — there is no shared loading across processes.
3. **Dead code dropped.** `showDatesFieldsOnChange` (`etfra-showDatesFields.js`) and `showHideDimensions`
   (`etfra-showHideDimensions.js`) are not bound to this process; they were not migrated. `showDatesFieldsOnChange`
   is the in-scope hook for D37588; migrate the others only when their owning process is in scope.
4. **`valueMap` + `refreshDisplayValue` collapsed to `setValueFromRecord`.** In the org callback, the
   classic three-step idiom (set value, inject label into `valueMap`, force `refreshDisplayValue()`) is a
   single `setValueFromRecord({ id, _identifier })`, which sets both the value and the displayed identifier
   for a value not in the loaded option list. Behaviorally equivalent.
5. **`resetValue()` → `clearValue()`.** The classic `documentNo.resetValue()` maps to the item proxy's
   `clearValue()` (the documented method for clearing a field value).
6. **Multi-selection value read made list-shape tolerant.** The classic `showDocumentNoField` treats the
   Document Type value as a SmartClient list (`documentTypeValues.length`, `.get(0)`). The migrated body
   reads `.length` directly and uses a small `getSelectedValue` helper that handles both a list with a
   `.get(...)` method and a plain JS array (`[0]`), so the single-selection check works regardless of the
   exact value shape the new selector returns. The behavior — "exactly one selected type that is in the
   allowlist" enables Document No., otherwise it is disabled and cleared — is unchanged. A leading
   `selectedTypes &&` null-guard is added so an empty/undefined value takes the else branch (disable +
   clear) rather than throwing, matching the classic intent.
7. **Null-guard added (org callback).** The org callback guards `getItem('GL_Journal_ID')`
   (`if (generalLedger)`) so a missing/renamed General Ledger field degrades gracefully instead of throwing.
   For the Document Type body, `documentNo` and `showDocumentNo` are not guarded because both are required
   parameters of this process (verified present); add guards only if those fields are ever removed.
8. **Empty-organization no-op preserved.** The classic guard `if (organization !== '')` is kept, so
   clearing Organization does not fire the handler — identical to Classic.
9. **Backend handler dependency.** The org flow calls the Java handler
   `com.etendoerp.financial.reports.advanced.handler.ChangeAcctSchemaActionHandler` (via
   `OB.RemoteCallManager.call` → `callAction`), exactly as Classic does. QA requires the module deployed
   and the handler returning `{ acctSchema: <id>, acctSchemaIdIdentifier: <label> }`. The Document Type
   flow has **no** backend dependency — it is pure client logic. Unchanged backend behavior, not a gap.
10. **No global namespace bootstrap needed.** The classic files start with `OB.ETFRA = {}` /
    `OB.ETFRA.OnChange = {}`. In the new UI each hook owns its scope, so no namespace self-registration is
    required for these self-contained onChange hooks; `em_etmeta_payscript_logic` stays empty.

## Manual-test checklist

Run in the new UI, on the **Journal Entries Report Advanced** process dialog.

### Organization → General Ledger (cluster-cloned hook)

1. **Open the dialog.** Confirm it opens with the standard parameter form (no custom component) and no
   console errors.
2. **Select an Organization.** Pick a value in **Organization**. Expect **General Ledger** to auto-populate
   with the accounting schema returned by the handler, showing the correct **label** (not a raw id).
3. **Change to a different Organization.** Expect the **General Ledger** value and its displayed label to
   update to the new organization's accounting schema.
4. **Clear the Organization** (set it to empty). Expect **no** call to the handler and **General Ledger**
   left unchanged (no error) — matching the classic empty-organization guard.
5. **Network check (optional).** On each non-empty Organization change, confirm a single POST to
   `ChangeAcctSchemaActionHandler` with `{ organization: <selected org id> }`, and that the UI applies the
   returned `acctSchema` / `acctSchemaIdIdentifier`.

### Document Type → Document No. / Show DocumentNo (process-unique hook)

6. **Select exactly one Document Type whose id is in the allowlist** (e.g. one of the ids in
   `DOCUMENT_TYPES_WITHOUT_DOC_NO`, such as a standard journal type). Expect: **Show DocumentNo** is set to
   **false/unchecked**, and **Document No.** becomes **enabled**.
7. **Select exactly one Document Type whose id is NOT in the allowlist.** Expect: **Show DocumentNo** is set
   to **true/checked**, **Document No.** becomes **disabled**, and any value previously typed in **Document
   No.** is **cleared**.
8. **Select two or more Document Types** (multi-selection). Expect the else branch: **Show DocumentNo**
   **true/checked**, **Document No.** **disabled** and **cleared** — because the rule requires exactly one
   selected type.
9. **Clear the Document Type** (no selection). Expect the else branch as well: **Document No.** disabled and
   cleared, no error.
10. **Toggle between an allowlisted single type and a non-allowlisted/multiple selection** a few times.
    Expect **Document No.** enable/disable and **Show DocumentNo** to flip correctly each time, with the
    field cleared whenever it becomes disabled.
11. **No console errors** during any of the above.
